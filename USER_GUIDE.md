# Memento 用户使用说明

> Memento 是一款端侧 AI Agent 应用，所有推理在你的手机本地完成，无需联网、无需账号、对话数据永不离开设备。
>
> 当前版本：v3.1 · 视觉模型：Qwen3.5-VL-0.8B（Q4_K_XL 量化） · 推理引擎：llama.cpp（mtmd 多模态）

## 1. 首次启动

首次打开 Memento 需要下载两个模型文件（共约 700MB）：

| 文件 | 大小 | 作用 |
|------|------|------|
| `mmproj-Qwen_Qwen3.5-0.8B-bf16.gguf` | ~200MB | 视觉编码器（处理图片/视频帧） |
| `Qwen3.5-0.8B-UD-Q4_K_XL.gguf` | ~500MB | 主语言模型（文字推理） |

**下载流程**：

1. 进入引导页，点击「开始下载」。
2. 应用会先下载视觉编码器，再下载主模型，下载完成后自动加载引擎。
3. 下载支持断点续传，若中断可重新进入应用继续。
4. 引擎初始化需要数秒（JNI 加载），请勿在此期间杀掉应用。
5. 加载完成后进入主聊天界面，即可开始对话。

> ⚠️ 下载需要稳定的网络环境（模型文件托管在阿里云 OSS）。建议在 Wi-Fi 下进行。

## 2. 基础对话

进入主界面后，底部输入栏可直接输入文字与 Memento 对话。

- **发送**：点击右侧发送按钮，或使用键盘发送键。
- **停止**：推理过程中发送按钮会变为停止按钮，点击可中断当前生成。
- **流式输出**：Memento 会逐字输出回复，无需等待整段完成。
- **主动搭话**：长时间未交互后回到应用，Memento 会主动问候。
- **记忆**：你的对话内容会以摘要形式存入本地记忆库，用于保持上下文连续性。可在设置中清除。

## 3. 多模态交互

Memento 支持图片、视频的输入与输出，所有处理均在端侧完成。

### 3.1 图片输入（多图）

Memento 支持一次发送最多 **10 张图片**，作为多模态上下文一起送入视觉模型推理。

#### 限制

| 项目 | 限制 |
|------|------|
| 单次最多图片数 | **10 张** |
| 图片总大小上限 | **50 MB** |
| 超限处理 | 弹出提示框，已选超限部分不会被添加 |

#### 操作步骤

1. 点击输入栏左侧的 **「+」** 按钮，弹出底部菜单。
2. 选择 **「图片（可多选）」**，打开系统相册。
3. 在相册中长按选中第一张，然后可继续点选多张（系统相册原生支持多选）。
4. 选完后返回聊天界面，所选图片会以缩略图 chip 横向排列在输入框上方，每张可单独删除。
5. **可选**：在输入框中输入说明文字（caption），例如「这是我家猫的三种睡姿」。
6. 点击发送按钮，图片连同说明一起送入模型。

#### 使用场景

- **多角度拍摄同一物体**：让模型综合多个视角理解（如前后左右四面）。
- **手动挑选视频关键帧**：比系统自动采样更可控——你可以从一段视频中截取关键画面，作为图片序列发送，模型即可理解「视频」内容。
- **前后对比图**：检测变化（如「装修前后」「妆容前后」）。
- **连环画/分镜**：发送一组连续画面，让模型讲述完整故事。

#### 说明文字的作用

- 若输入了说明文字，模型会以该文字作为理解图片的引导（如「找出这两张图的不同点」）。
- 若未输入说明，模型默认执行「请描述这些图片」。

> 💡 选图后图片与视频互斥：选择图片会清空已选视频，反之亦然。

### 3.2 视频输入（单选）

Memento 支持发送单个短视频，系统会自动采样视频关键帧送入模型。

#### 限制

| 项目 | 限制 |
|------|------|
| 视频数量 | **单个**（不支持多选） |
| 视频大小上限 | **50 MB** |
| 采样策略 | 前 5 秒，每秒 2 帧（共约 10 帧） |
| 超限处理 | 弹出提示框，附件不会被添加 |

#### 操作步骤

1. 点击输入栏 **「+」** → 选择 **「视频」**。
2. 在系统选择器中挑选一个视频。
3. 选完后返回聊天界面，视频以单 chip 形式显示在输入框上方。
4. **可选**：输入说明文字（如「这是我家狗学接飞盘的过程」）。
5. 点击发送，系统会：
   - 校验视频大小；
   - 使用 `MediaMetadataRetriever` 提取前 5 秒关键帧；
   - 将帧压缩为 JPEG 后作为图片序列送入模型。

#### 与多图输入的区别

| 维度 | 视频输入 | 多图输入 |
|------|---------|---------|
| 来源 | 系统视频选择器 | 系统相册多选 |
| 帧挑选 | 系统自动采样前 5s | 用户手动挑选 |
| 控制度 | 低（固定采样） | 高（用户决定） |
| 适用 | 完整视频片段理解 | 精心挑选的关键画面 |

> 💡 想要更精确地控制模型看到的画面？推荐使用「多图输入」手动挑选关键帧，效果往往优于系统自动采样。

### 3.3 图片输出

Memento 可以根据你的文字描述在端侧生成图片，支持写实、插画、矢量等多种风格。

#### 触发方式

直接在对话中用自然语言要求生成图片即可，例如：

- 「画一只戴帽子的猫」
- 「帮我生成一张生日贺卡」
- 「画一幅日落海景，写实风格」
- 「设计一个极简风格的 App 图标」

#### 工作原理

1. Memento 识别到生成意图后，会在回复首行输出 `[GEN_IMAGE:图片主题描述]` 标记。
2. 应用解析该标记，调用端侧图像生成管线渲染图片。
3. 生成完成后，图片会以附件形式显示在助手回复下方，点击可查看大图。

#### 风格指引

| 风格 | 适用场景 | 描述示例 |
|------|---------|---------|
| 写实 | 风景、静物、场景概念图 | 「写实风格的山间日出」 |
| 插画 | 贺卡、海报、故事插图 | 「温暖插画风格的生日贺卡」 |
| 矢量 | 图标、Logo、几何图形 | 「极简矢量的飞机图标」 |
| 暗色 | 科技感、夜景、霓虹 | 「暗色调赛博朋克城市」 |

> 💡 描述越具体，生成结果越贴近预期。建议包含主体、风格、氛围三个要素。

### 3.4 视频输出

Memento 可以根据文字描述逐帧渲染生成视频，输出为 MP4 文件。

#### 触发方式

- 「帮我做一个生日快乐的视频」
- 「生成一段新年祝福动画」
- 「做一个极简风格的加载动画」

#### 工作原理

1. 模型输出 `[GEN_VIDEO:视频主题描述]` 标记。
2. 应用调用端侧逐帧渲染管线：根据主题生成每一帧画面，再合成为视频。
3. 渲染完成后生成 MP4 文件，显示在助手回复中，可直接播放。

#### 适用场景

- 节日贺卡与祝福视频
- 品牌加载动画与过渡动效
- 短主题动画与分镜演示
- 社交媒体短视频素材

> 💡 视频生成是逐帧渲染过程，时长越长耗时越久。建议从短主题（5 秒内）开始尝试。

## 4. Skill 市场

Memento 采用「核心 + Skill」的模块化架构：

- **核心**：推理引擎 + 沙箱运行时 + Skill 注册表，保持精简。
- **Skill**：可插拔的能力扩展包，按需安装，沙箱内运行，不调用即静默。

### 4.1 Skill 的特点

| 特性 | 说明 |
|------|------|
| 沙箱执行 | Skill 运行在隔离沙箱内，无法访问核心数据与系统权限 |
| 按需激活 | 不主动调用时完全静默，不消耗资源 |
| 扫码安装 | 通过网站端 Skill 市场扫码即可安装到本地 |
| 可随时卸载 | 在设置中可查看已安装 Skill 并卸载 |

### 4.2 网站端 Skill 市场

访问 Memento 官方网站（https://memento.app）的 **Skill 市场** 页面，可浏览所有可用 Skill。

**安装流程**：

1. 在网站 Skill 市场中找到想要的 Skill，点击「安装到 Memento」。
2. 网站生成一个二维码。
3. 打开手机端 Memento → 设置 → Skill 管理 → 扫一扫。
4. 扫码后确认安装，Skill 包（ZIP 格式）下载并解压到本地沙箱目录。
5. 安装完成后，Skill 自动注册到核心，可在对话中按需调用。

### 4.3 以「漫剧 Skill」为例

**漫剧 Skill** 是 Memento Skill 市场的首发能力模块，本质上是一座完整的端侧漫剧加工厂。

#### 它做什么

漫剧 Skill 不是简单的"AI 画漫画"，而是一条完整的工业化创作工作流：

```
故事大纲 → 剧本拆解 → 角色设计 → 分镜脚本 → 逐帧渲染 → 排版合成 → 漫剧成片
```

| 环节 | Skill 工作 |
|------|-----------|
| 剧本拆解 | 将你的故事大纲拆解为可拍摄的场景与对白 |
| 角色设计 | 自动设定角色形象，并在所有分镜中保持一致 |
| 分镜脚本 | 规划每个画面的构图、机位、对白气泡 |
| 逐帧渲染 | 按分镜逐帧生成漫画风格画面（写实/半写实/日漫/美漫可选） |
| 排版合成 | 自动排版分格、添加对白气泡、生成封面与封底 |
| 输出 | 输出可分享的漫剧图集或长图 |

#### 使用方式

安装后直接对话触发：

- 「帮我做一个关于小猫探险的 8 格漫剧」
- 「把这段故事做成漫剧：……」
- 「画一个赛博朋克风格的短篇漫剧，主角是侦探」

Skill 会自主完成全流程，过程中可随时介入调整（如「第 3 格换成特写」「对白改成中文」）。

#### 为什么做成 Skill

- 漫剧加工厂链路长、依赖多，作为核心常驻会拖累主对话性能；
- 不是所有用户都需要，按需安装让核心保持精简；
- 沙箱运行确保即使工作流出错也不影响主对话稳定性；
- Skill 可独立迭代升级，新画风、新排版模板无需更新核心 App。

#### 安装步骤

1. 访问网站 Skill 市场，找到「漫剧加工厂」。
2. 点击「安装」，用手机 Memento 扫描二维码。
3. 安装完成后，在对话中说出创作需求即可触发。
4. 不再需要时，前往 设置 → Skill 管理 → 漫剧加工厂 → 卸载。

> 💡 Skill 市场会持续更新，未来会有更多能力（绘画增强、文档处理、代码助手等）以 Skill 形式发布。

## 5. 设置与隐私

### 设置项

| 设置 | 说明 |
|------|------|
| 主题模式 | 浅色 / 深色 / 跟随系统 |
| 外观皮肤 | 多种皮肤可选 |
| 视频画质 | 影响视频输出的分辨率与帧率 |
| Skill 管理 | 查看已安装 Skill、卸载、扫码安装新 Skill |
| 清除聊天记录 | 清空当前对话历史 |
| 清除所有记忆 | 清空本地记忆库（不可恢复） |
| 卸载模型 | 释放内存，下次推理时自动重新加载 |

### 隐私

- ✅ 所有推理在端侧完成，对话内容不上传任何服务器。
- ✅ 模型文件从阿里云 OSS 下载后离线运行。
- ✅ 记忆库存储在本地加密数据库。
- ✅ 图片/视频处理在本地完成，不上传。
- ⚠️ 仅在首次下载模型、访问 Skill 市场时需要网络。

## 6. 常见问题

### Q: 下载模型时提示「403」或「下载中断」？

A: 检查网络环境。模型托管在阿里云 OSS，部分网络可能需要科学上网。下载支持断点续传，重新进入应用会自动继续。

### Q: 引擎初始化时崩溃？

A: 通常是模型文件不完整或设备内存不足。请前往设置 → 卸载模型，重新下载。低端设备（< 6GB RAM）可能无法稳定运行。

### Q: 发送图片后提示「图片处理失败」？

A: 可能原因：
- 单张图片过大（建议 < 10MB）；
- 图片格式不被系统支持；
- 设备内存不足。建议选择较小的图片或减少一次发送的数量。

### Q: 视频输入后模型看不到内容？

A: 视频输入依赖系统采样前 5 秒关键帧。若视频前 5 秒是黑屏或无关内容，模型理解会受限。建议：
- 拍摄时让关键内容出现在前 5 秒；
- 或改用「多图输入」手动挑选关键帧。

### Q: 生成的图片/视频效果不理想？

A: 生成结果与描述的具体程度强相关。建议：
- 描述时明确主体、风格、氛围三要素（如「写实风格的山间日出，暖色调」）；
- 指定画风（写实 / 插画 / 矢量 / 暗色）；
- 视频建议从短主题（5 秒内）开始，逐步迭代；
- 多次尝试不同描述措辞，模型对表述敏感。

### Q: Skill 安装后没有反应？

A: Skill 在沙箱中运行，只有当对话内容匹配其触发条件时才会激活。可尝试用更明确的指令触发（如「画一个漫剧」触发漫剧 Skill）。

### Q: 应用占用内存很大？

A: 模型加载后会常驻内存（约 600MB-1GB），这是端侧推理的必要开销。如需释放，可在设置中「卸载模型」，下次对话时会自动重新加载。
