## 1. 首次启动与密钥配置

v4.0 起所有智能能力由云端 API 提供，首次使用前需配置 API 密钥。

### 密钥模式

Memento 支持两种密钥模式（设置 → API 密钥）：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **代理模式（PROXY）** | 走官方代理端点，使用统一代理 Token，无需自备 API Key | 官方提供代理服务时 |
| **直连模式（DIRECT）** | 用户自填 OpenAI / Kling API Key，请求直连官方端点 | 已有 OpenAI / Kling 账号 |

### 首次启动流程

1. 启动应用 → 引导页（品牌展示）。
2. 进入激活页（如需激活码）。
3. 进入 Onboarding 页 → 提示配置 API 密钥。
4. 配置完成后进入主聊天界面，即可开始对话。

> ⚠️ 直连模式下，请确保你的 API Key 有 GPT-4o、DALL-E 3、Kling 的调用配额。

## 2. 基础对话

进入主界面后，底部输入栏可直接输入文字与 Memento 对话。

- **发送**：点击右侧发送按钮。
- **停止**：生成过程中发送按钮会变为停止按钮，点击可中断当前流式输出。
- **流式输出**：GPT-4o 通过 SSE 流式返回，逐字输出回复。
- **主动搭话**：长时间未交互后回到应用，Memento 会主动问候。
- **记忆**：你的对话摘要会存入本地记忆库（Room SQLite），用于保持上下文连续性。可在设置中清除。
- **历史上下文**：每次请求会携带最近 10 条对话历史，超出部分由本地记忆摘要补全。

## 3. 多模态交互

Memento 是原生多模态架构，图片、视频、文字在输入端地位平等，均可作为生成的参考素材。理解与生成全部由云端 API 完成。

### 3.0 架构原理：理解 → 标记 → 云端渲染

Memento v4.0 的多模态生成采用**「理解 → 标记 → 云端渲染」**架构：

```
输入（自然语言 / 图片 / 视频）
    ↓
GPT-4o 多模态理解（看懂你的输入 + 决定是否生成媒体）
    ↓
GPT-4o 输出生成标记 + 媒体描述 prompt
  · [GEN_IMAGE] English prompt for DALL-E 3
  · [GEN_VIDEO] 中文视频描述 for Kling
  · [EDIT_IMAGE] 编辑后完整画面描述
    ↓
应用解析标记 → 调用对应云端 API 渲染
  · DALL-E 3 生成/编辑图片
  · 可灵 Kling 图生视频 / 文生视频
    ↓
输出（图片 / 视频）
```

#### 为什么这样设计

让 GPT-4o 专注「理解 + 翻译为生成 prompt」，让 DALL-E 3 / Kling 专注「渲染」，三者各司其职：

| 优势 | 说明 |
|------|------|
| **质量生产级** | DALL-E 3 / Kling 是业界顶尖的生成模型，质量远超端侧轻量模型 |
| **多模态理解强** | GPT-4o 同时具备顶级图片理解 + 对话能力 |
| **结构可控** | GPT-4o 通过标记决定何时生成、生成什么，避免误触发 |
| **多模态输入** | 输入不限于文字，可用图片/视频作为参考素材 |

#### 三种生成标记

GPT-4o 在回复中按需输出以下标记：

| 标记 | 含义 | 触发的 API |
|------|------|-----------|
| `[GEN_IMAGE]` | 生成新图片 | DALL-E 3（英文 prompt） |
| `[GEN_VIDEO]` | 生成新视频 | 可灵 Kling（中文 prompt） |
| `[EDIT_IMAGE]` | 编辑最近生成的图片 | DALL-E 3（编辑后完整画面描述） |

> 💡 标记内容由 GPT-4o 自动产出，用户无需手动输入。只需用自然语言表达需求即可。

### 3.1 三段式工作流：帧 / 图形 / 合成

Memento 的多模态输入采用**三段式工作流**，通过「+」菜单进入：

```
帧   ─→ 1s 视频 → 24 帧关键帧 → 保存相册 + 进缓存
  │
  ↓ 关键帧进入"图形"工坊
图形 ─→ 元素替换对话框（用户元素+原图元素+关键帧）
  │     用户指令改造特定元素（替换/调整/添加）
  │     输出：批量修改后的关键帧图片 → 保存进缓存
  ↓ 改造后的关键帧进入"合成"
合成 ─→ 取当前缓存的关键帧序列
       可灵 Kling 图生视频 → MP4
       输出：MP4 视频
```

**三段式通过 KeyFrameStore 缓存传递关键帧状态**，让用户可以分步骤完成复杂任务，而不是一次性把所有素材塞给模型。

#### 加号菜单

点击输入栏左侧的 **「+」** 按钮，弹出三个选项：

| 选项 | 图标 | 功能 | 前置条件 |
|------|------|------|---------|
| **帧** | 闪光 | 选择 1s 视频 → 提取 24 帧关键帧 → 保存相册 + 进缓存 | 无 |
| **图形** | 图片 | 元素替换对话框，改造关键帧输出图片 | 有关键帧缓存时体验最佳 |
| **合成** | 胶片 | 将缓存关键帧通过可灵 Kling 合成为 MP4 视频 | 缓存必须有关键帧 |

> 💡 输入栏上方会显示关键帧缓存指示器：`🎬 关键帧缓存：N 帧（来源）`，可一键清空。

### 3.2 帧：1s 视频 → 24 帧关键帧

点击「+」→ **帧**，打开**「帧工坊：1s 视频 → 24 帧关键帧」**对话框。

#### 功能

- **选择 1s 视频**：从系统选择器挑选一个视频（≤1 秒，≤50MB）。
- **提取 24 帧**：系统使用 `MediaMetadataRetriever` 提取 24 帧关键帧（256×256 缩略图）。
- **保存到相册**：提取的帧自动保存到相册的 `Pictures/Memento` 目录。
- **进入缓存**：帧 Uri 自动进入 KeyFrameStore，供「图形」「合成」使用。

#### 限制

| 项目 | 限制 |
|------|------|
| 视频时长 | **≤ 1 秒**（超过部分不采样） |
| 视频大小 | **≤ 50 MB** |
| 采样帧率 | **24 fps**（1 秒 × 24 帧 = 24 帧缩略图） |
| 帧尺寸 | **256×256 缩略图** |
| 超限处理 | 弹出提示框，附件不会被添加 |

#### 操作步骤

1. 点击「+」→ **帧**，打开帧工坊对话框。
2. 点击「选择 1s 视频」，从系统选择器挑选视频。
3. 系统校验大小（≤50MB），超限弹出提示。
4. 提取进度实时显示：「正在提取关键帧... N/24」。
5. 提取完成后，帧保存到相册并进入缓存。
6. 关闭对话框，可在「图形」中改造，或在「合成」中生成 MP4。

#### 为什么是 1 秒 24 帧

- **1 秒**：控制上传体积与云端处理延迟，1 秒是平衡点。
- **24 帧**：达到视频最低帧率标准，可灵 Kling 能理解连续动作。
- **256×256 缩略图**：减小上传体积，让模型聚焦于内容理解而非细节。

### 3.3 图形：元素替换改造关键帧

点击「+」→ **图形**，打开**「图形工坊：元素替换改造关键帧」**对话框。

#### 对话框布局

```
┌─────────────────────────────────────────┐
│  图形工坊：元素替换改造关键帧           │
├─────────────────────────────────────────┤
│  [+ 用户元素] [+ 原图元素] → [+ 关键帧] │
│    缩略图       缩略图       缩略图×N   │
│                                         │
│  已选关键帧 N 张（来自帧工坊缓存或手动选择）│
│  [缩略图列表，可单独删除]               │
│                                         │
│  选择用户元素和原图元素，系统将替换关键  │
│  帧中的对应元素，输出改造后的关键帧图   │
│  片进入缓存，供「合成」生成 MP4。       │
├─────────────────────────────────────────┤
│              [取消]  [提交]             │
└─────────────────────────────────────────┘
```

#### 三个选择入口

| 入口 | 数量 | 用途 |
|------|------|------|
| 用户元素 | 1 张 | 要替换进去的素材 |
| 原图元素 | 1 张 | 原图中要被替换的部分 |
| 关键帧 | 4-12 张 | 动画基础，自动从 KeyFrameStore 预填 |

#### 关键帧自动预填

打开图形工坊时，若 KeyFrameStore 中已有缓存的关键帧（来自帧工坊），会自动填入「关键帧」入口，无需重复选择。

#### 提交后发生什么

1. 三类素材合并为图片列表，显示为输入框上方的 chip。
2. **若有关键帧**：同步到 KeyFrameStore，供「合成」使用。
3. 在输入框输入说明文字（可选），点击发送。
4. GPT-4o 理解图片内容，输出 `[GEN_IMAGE]` 标记 + DALL-E 3 prompt，由 DALL-E 3 生成改造后的关键帧图片进入缓存。

#### 使用场景

- **元素替换**：「把原图中的猫替换成我的猫」——用户元素=我的猫，原图元素=原图中的猫。
- **批量改造关键帧**：基于帧工坊提取的 24 帧，把某一元素替换为用户元素，生成改造后的图片序列。
- **纯参考图**：只选一张用户元素作为风格参考，配合文字描述生成图片。

### 3.4 合成：关键帧 → MP4

点击「+」→ **合成**，触发视频合成流程。

#### 前置条件

- KeyFrameStore 中必须有关键帧（否则菜单项禁用并提示）。

#### 合成流程

1. 应用取 KeyFrameStore 中所有关键帧图片 Uri。
2. 关键帧作为图片序列，调用可灵 Kling 图生视频 API。
3. Kling 异步处理：POST 创建任务 → 轮询任务状态（每 3 秒）→ 直到 `succeed` 或 `failed`。
4. 任务成功后下载 MP4 视频文件。
5. 进度实时显示在对话气泡中（提交中 → 处理中 → 完成）。

#### 输出规格

| 项目 | 规格 |
|------|------|
| 输出格式 | MP4 |
| 模型 | 可灵 Kling v2（std 模式） |
| 任务超时 | 180 秒 |
| 轮询间隔 | 3 秒 |
| 视频时长 | 由 Kling 根据输入帧数决定 |

> 💡 合成是整个三段式工作流的终点。帧工坊提取关键帧，图形工坊改造关键帧，合成产出 MP4。三段可灵活组合，也可单独使用。

### 3.5 视频输入（1 秒 24 帧）

Memento 支持发送单个短视频，系统会采样视频关键帧送入 GPT-4o。视频提取的帧也会自动进入 KeyFrameStore，可在「帧」「合成」中继续使用。

#### 限制

| 项目 | 限制 |
|------|------|
| 视频数量 | **单个**（不支持多选） |
| 视频大小上限 | **50 MB** |
| 视频时长 | **前 1 秒**（超过部分不采样） |
| 采样帧率 | **24 fps**（1 秒 × 24 帧 = 24 帧缩略图） |
| 帧尺寸 | **256×256 缩略图**（减小上传体积） |
| 超限处理 | 弹出提示框，附件不会被添加 |

#### 操作步骤

1. 点击输入栏 **「+」** → 选择 **「视频」**。
2. 在系统选择器中挑选一个视频。
3. 选完后返回聊天界面，视频以单 chip 形式显示在输入框上方。
4. **可选**：输入说明文字（如「这是我家狗学接飞盘的过程」）。
5. 点击发送，系统会：
   - 校验视频大小；
   - 使用 `MediaMetadataRetriever` 提取前 1 秒、24 帧关键帧；
   - 将每帧压缩为 256×256 缩略图 JPEG；
   - 作为图片序列上传给 GPT-4o 理解。

> 💡 想要更精确地控制模型看到的画面？推荐使用「图形」工坊手动挑选关键帧，效果往往优于系统自动采样。

### 3.6 图片输出（DALL-E 3）

Memento 的图片生成由 DALL-E 3 完成，GPT-4o 负责理解需求并输出英文 prompt。

#### 核心机制

```
输入（自然语言 / 图片 / 视频）
    ↓
GPT-4o 理解 + 输出 [GEN_IMAGE] + 英文 DALL-E 3 prompt
    ↓
应用解析标记 → 调用 DALL-E 3 API
    ↓
DALL-E 3 生成图片 → 保存为 PNG
```

#### 输出规格

| 项目 | 规格 |
|------|------|
| 输出格式 | PNG |
| 生成模型 | DALL-E 3 |
| Prompt 语言 | 英文（由 GPT-4o 自动翻译） |

#### 触发方式

直接在对话中表达需求即可，输入形式灵活：

- **纯文字**：「画一只戴帽子的猫」
- **文字 + 参考图**：先发送一张参考图，再说「按这个风格画一张生日贺卡」
- **多图参考**：发送多张产品图，说「为这个产品设计一张海报」

#### 风格指引

DALL-E 3 支持丰富的风格，描述时可指定：

| 风格 | 适用场景 | 示例描述 |
|------|---------|---------|
| 写实照片 | 风景、静物、人像 | 「画一幅日落海景，照片级写实」 |
| 插画 | 贺卡、海报、故事插图 | 「画一张温暖风格的生日贺卡插画」 |
| 3D 渲染 | 产品、概念图、品牌 | 「设计一个 3D 渲染的飞机图标」 |
| 油画/水彩 | 艺术创作 | 「画一幅水彩风格的城市夜景」 |

> 💡 DALL-E 3 可生成照片级写实图像，也可生成插画、3D、油画等多种风格，描述时直接说明想要的风格即可。

### 3.7 视频输出（可灵 Kling）

Memento 的视频生成由可灵 Kling 完成，GPT-4o 负责理解需求并输出中文视频描述。

#### 核心机制

```
输入（自然语言 / 图片 / 视频）
    ↓
GPT-4o 理解 + 输出 [GEN_VIDEO] + 中文 Kling prompt
    ↓
应用解析标记 → 调用 Kling API（图生视频 或 文生视频）
    ↓
异步轮询任务状态（每 3 秒）→ 下载 MP4
```

#### 输出规格

| 项目 | 规格 |
|------|------|
| 输出格式 | MP4 |
| 生成模型 | 可灵 Kling v2（std 模式） |
| 任务超时 | 180 秒 |
| 轮询间隔 | 3 秒 |
| 任务状态 | submitted → processing → succeed / failed |

#### 触发方式

- **纯文字**：「帮我做一个生日快乐的动画」
- **文字 + 参考图**：发送一张 Logo，说「为这个 Logo 做一个加载动画」
- **文字 + 参考视频**：发送一段 1s 视频，说「按这个风格做一段动画」
- **三段式合成**：通过「图形→帧→合成」工作流产出（见 3.1-3.4）

#### 渲染过程

1. GPT-4o 输出 `[GEN_VIDEO]` 标记 + 中文视频描述。
2. 应用解析标记：
   - 若有参考图片 → 调用 Kling 图生视频（image-to-video）
   - 若纯文字 → 调用 Kling 文生视频（text-to-video）
3. Kling 异步处理：
   - POST 创建任务 → 返回 task_id
   - 每 3 秒 GET 轮询任务状态
   - 状态变为 `succeed` 后下载 MP4
   - 超时 180 秒或状态变为 `failed` 时报错
4. 下载完成后视频显示在对话气泡中。

#### 适用场景

- 节日贺卡与祝福动画
- 品牌加载动画与过渡动效
- 短主题动画与分镜演示
- 社交媒体短视频素材

> 💡 视频生成是异步过程，Kling 通常需要数十秒到数分钟。请保持应用在前台或网络畅通，超时会自动报错。

### 3.8 图片编辑（自然语言编辑）

这是「理解 → 标记 → 云端渲染」架构的核心优势——**自然语言编辑**。

#### 工作原理

当你对已生成的图片提出修改要求时，应用会：

1. 自动缓存最近一次生成的图片路径。
2. GPT-4o 检测到编辑意图，输出 `[EDIT_IMAGE]` 标记 + 编辑后的完整画面描述（英文）。
3. 应用将编辑后的完整画面描述送给 DALL-E 3 重新生成。
4. DALL-E 3 输出修改后的图片。

#### 操作示例

```
你：画一只橙色的猫
Memento：[GEN_IMAGE] → DALL-E 3 生成了一张橙色猫的图片

你：把猫换成黑色
Memento：[EDIT_IMAGE] → DALL-E 3 基于编辑描述重新生成黑色猫

你：去掉背景，换成夜晚
Memento：[EDIT_IMAGE] → DALL-E 3 重新生成夜晚背景的黑色猫

你：猫的位置往左移一点
Memento：[EDIT_IMAGE] → DALL-E 3 重新生成左移后的图片
```

#### 支持的编辑指令

| 编辑类型 | 示例 |
|---------|------|
| 颜色替换 | 「把猫换成黑色」「背景改成蓝色」 |
| 元素删除 | 「去掉太阳」「删除背景里的树」 |
| 位置调整 | 「猫往左移」「太阳放高一点」 |
| 大小调整 | 「猫放大一点」「太阳缩小」 |
| 元素添加 | 「加一个月亮」「天上加几朵云」 |
| 风格转换 | 「改成水彩风格」「换成 3D 渲染」 |

> 💡 编辑模式由 GPT-4o 自动识别（无需特殊指令），只要表达修改意图且有最近生成的图片即可。DALL-E 3 会基于编辑后的完整画面描述重新生成，比像素级修改更自然。

## 4. Skill 市场

Memento 采用「核心 + Skill」的模块化架构：

- **核心**：云端 API 客户端 + 沙箱运行时 + Skill 注册表，保持精简。
- **Skill**：可插拔的能力扩展包，按需安装，沙箱内运行，不调用即静默。

### 4.1 Skill 的特点

| 特性 | 说明 |
|------|------|
| 沙箱执行 | Skill 运行在隔离沙箱内，无法访问核心数据与系统权限 |
| 按需激活 | 不主动调用时完全静默，不消耗资源 |
| 扫码安装 | 通过网站端 Skill 市场扫码即可安装到本地 |
| 可随时卸载 | 在设置中可查看已安装 Skill 并卸载 |

### 4.2 网站端 Skill 市场

访问 Memento 官方网站的 **Skill 市场** 页面，可浏览所有可用 Skill。

**安装流程**：

1. 在网站 Skill 市场中找到想要的 Skill，点击「安装到 Memento」。
2. 网站生成一个二维码。
3. 打开手机端 Memento → 设置 → Skill 管理 → 扫一扫。
4. 扫码后确认安装，Skill 包（ZIP 格式）下载并解压到本地沙箱目录。
5. 安装完成后，Skill 自动注册到核心，可在对话中按需调用。

### 4.3 以「漫剧 Skill」为例

**漫剧 Skill** 是 Memento Skill 市场的首发能力模块，本质上是一座完整的云端漫剧加工厂。

#### 它做什么

漫剧 Skill 是一条完整的工业化创作工作流：

```
故事大纲 → 剧本拆解 → 角色设计 → 分镜脚本 → 逐帧生成 → 排版合成 → 漫剧成片
```

| 环节 | Skill 工作 |
|------|-----------|
| 剧本拆解 | GPT-4o 将你的故事大纲拆解为可拍摄的场景与对白 |
| 角色设计 | GPT-4o 设定角色形象，DALL-E 3 生成角色设定图 |
| 分镜脚本 | GPT-4o 规划每个画面的构图、机位、对白气泡 |
| 逐帧生成 | DALL-E 3 按分镜逐帧生成漫画风格画面 |
| 排版合成 | 自动排版分格、添加对白气泡、生成封面与封底 |
| 输出 | 输出可分享的漫剧图集或长图 |

#### 为什么漫剧 Skill 适合云端架构

漫剧加工厂的核心需求是**角色一致性**——同一个角色在所有分镜中必须长相一致。云端架构的优势：

- DALL-E 3 生成质量远超端侧模型，画面表现力强
- GPT-4o 能精确理解角色描述，确保 DALL-E 3 prompt 一致
- 可灵 Kling 可基于关键帧生成过渡动画

#### 使用方式

安装后直接对话触发：

- 「帮我做一个关于小猫探险的 8 格漫剧」
- 「把这段故事做成漫剧：……」
- 「画一个赛博朋克风格的短篇漫剧，主角是侦探」

Skill 会自主完成全流程，过程中可随时介入调整（如「第 3 格换成特写」「对白改成中文」）。

#### 安装步骤

1. 访问网站 Skill 市场，找到「漫剧加工厂」。
2. 点击「安装」，用手机 Memento 扫描二维码。
3. 安装完成后，在对话中说出创作需求即可触发。
4. 不再需要时，前往 设置 → Skill 管理 → 漫剧加工厂 → 卸载。

> 💡 Skill 市场会持续更新，未来会有更多能力（绘画增强、文档处理、代码助手等）以 Skill 形式发布。

### 设置项

### 隐私

- ⚠️ 对话内容、图片、视频会上传到云端 API（GPT-4o / DALL-E 3 / Kling）进行处理。
- ✅ 记忆库存储在本地加密数据库（Room SQLite + EncryptedSharedPreferences）。

- ⚠️ 代理模式下，请求经官方代理端点转发；直连模式下，请求直连 OpenAI / Kling 官方端点。

> 💡 如对数据隐私有严格要求，请使用直连模式并自行管理 API Key，或避免发送敏感内容。

## 6. 常见问题

### Q: 启动后无法对话，提示 API 错误？

A: 检查 API 密钥配置（设置 → API 密钥）：
- **代理模式**：确认代理 Token 有效，代理服务可用。
- **直连模式**：确认 API Key 正确，且有 GPT-4o / DALL-E 3 / Kling 的调用配额。
- 检查网络连接。

### Q: 发送图片后提示「图片处理失败」？

A: 可能原因：
- 单张图片过大（建议 < 10MB）；
- 图片格式不被系统支持；
- 设备内存不足。建议选择较小的图片或减少一次发送的数量。

### Q: 视频输入后模型看不到内容？

A: 视频输入只采样前 **1 秒** 的 24 帧。若视频前 1 秒是黑屏或无关内容，模型理解会受限。建议：
- 拍摄/剪辑时让关键内容出现在前 1 秒；
- 或改用「多图输入」手动挑选关键帧。

### Q: 图片生成失败？

A: 可能原因：
- DALL-E 3 API 配额不足（直连模式）；
- 网络不稳定导致请求超时；
- GPT-4o 未输出 `[GEN_IMAGE]` 标记（请更明确地表达生成需求）。

建议重试，或检查 API Key 配额。

### Q: 视频生成一直显示「处理中」？

A: 可灵 Kling 是异步任务，通常需要数十秒到数分钟。Memento 每 3 秒轮询一次任务状态，超时为 180 秒。若超时：
- 检查网络连接；
- 稍后重试（Kling 服务可能繁忙）；
- 简化视频描述。

### Q: 编辑指令没生效？

A: 编辑模式由 GPT-4o 自动识别，需要满足：
1. 表达了修改意图（修改/替换/去掉/换成/改成/调整/edit/modify/replace/remove）；
2. 有最近一次生成的图片（应用会缓存其路径）。

若刚清空过对话或重启过应用，缓存的图片路径可能已丢失，需要重新生成图片后再编辑。

### Q: 视频合成失败？

A: 可能原因：
- KeyFrameStore 中没有关键帧（需先通过「帧」或「图形」生成关键帧）；
- 可灵 Kling API 配额不足；
- 关键帧图片格式不支持；
- 网络不稳定。

建议检查关键帧缓存指示器，确认有缓存帧后再合成。

### Q: Skill 安装后没有反应？

A: Skill 在沙箱中运行，只有当对话内容匹配其触发条件时才会激活。可尝试用更明确的指令触发（如「画一个漫剧」触发漫剧 Skill）。

### Q: 代理模式和直连模式有什么区别？

A:
- **代理模式**：使用官方代理端点，统一代理 Token，无需自备 API Key。适合官方提供代理服务的场景。
- **直连模式**：用户自填 OpenAI / Kling API Key，请求直连官方端点。适合已有 OpenAI / Kling 账号的用户，可自行控制配额与账单。

两种模式下功能完全一致，仅密钥来源与请求路径不同。
